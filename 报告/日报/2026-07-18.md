## 🔌 SerDes与互连技术

1. 🔥 **OIF CEI-224G-MR/Linear 项目继续把 224G 电接口拆成不同 reach 与线性光模块方向。** OIF current work 页面列出 CEI-224G-MR、CEI-224G-Linear 等项目，MR 面向 chip-to-chip 互连，Linear 面向 224G full linear optical modules；对 1.6T 模块 host SerDes、LPO/LRO 和 retimer 放置仍是第一优先级标准入口。来源：[OIF Current Work](https://www.oiforum.com/technical-work/current-work/)
2. 🔥 **IEEE P802.3dj 仍是 200G/lane、800GbE/1.6TbE 规格收敛的主线。** 802.3dj 公开页面和近期议程显示 July 2026 plenary 已在 Montreal 周完成，后续 September interim 继续推进；工程上应继续跟踪 optical/electrical ad hoc 的 baseline、comment resolution 和 FEC 相关决议。来源：[IEEE P802.3dj](https://www.ieee802.org/3/dj/index.html)、[IEEE 802.3 interim sessions](https://www.ieee802.org/3/interims/index.html)
3. 🔥 **448G CEI 处于可行性、调制与测试能力共同讨论阶段。** Ethernet Alliance/OIF 448G panel 材料强调 3.2Tbps 级信号生成和采集测试仪器已存在，但未来高速 transceiver/interconnect 仍可能在调制格式、reach、equalization 与功耗之间出现多路线竞争。来源：[Ethernet Alliance OIF 448G panel PDF](https://ethernetalliance.org/wp-content/uploads/2025/12/OIF_TEF_Panel2_2512.pdf)
4. ⭐ **Marvell 3nm 224G LR SerDes + co-packaged copper demo 把 scale-up/scale-inside 铜互连距离推到系统级验证。** Marvell 博文称其 224G LR SerDes 在 OFC 2026 与 Luxshare-Tech 方案中驱动 2.5 m CPC-backplane-CPC 通道，重点不是替代光，而是为 tray 内、rack 内短距高带宽互连提供铜与光之间的工程折中。来源：[Marvell 224G Long-Range SerDes](https://www.marvell.com/blogs/224g-long-range-serdes-scale-up-scale-inside.html)
5. ⭐ **Synopsys 224G Ethernet PHY IP 明确支持 UALink 200G 与 1.6T Ethernet 系统集成。** 官方产品页称该 PHY 覆盖 1.25-224Gbps、PAM4/NRZ，并面向 IEEE/OIF 电规范；对自研 AI ASIC、switch ASIC 和 NIC 来说，SerDes IP 已经与协议控制器、验证 IP 和封装 channel model 绑定。来源：[Synopsys 224G Ethernet PHY IP](https://www.synopsys.com/designware-ip/interface-ip/ethernet/224g-ethernet-phy-ip.html)

## 🔦 光学与光模块

1. 🔥 **OIF 在 OFC 2026 的 multi-vendor demo 已把 CEI-448G、CEI-224G、CMIS、co-packaging 和 EEI 放到同一互操作框架。** 这说明下一代 AI 光互连不只是单模块速率升级，而是电接口、模块管理、线性光、节能接口和 CPO 共同收敛。来源：[OIF @ OFC 2026](https://www.oiforum.com/meetings-events/oif-ofc-2026/)
2. 🔥 **OFC 2026 趋势材料继续把 1.6T/3.2T 模块、CPO、silicon photonics 和 optical I/O 作为 AI data center 主轴。** 该会议已结束，但其公开 program/产业总结仍是回查短距光互连论文与演示的核心索引；后续日报应优先跟踪可落地的 200G/lane、400G/lane 和 packaging 证据。来源：[OFC 2026 Outlook](https://tspasemiconductor.substack.com/p/ofc-2026-outlook-ai-data-center-optical)、[Optica OFC 2026](https://opg.optica.org/conference.cfm?meetingid=5)
3. ⭐ **Marvell 扩展 1.6T optical DSP portfolio，Ara T、Ara X 与 Petra 分别指向 TRO、可靠性增强和 8x100G-to-4x200G gearbox。** 官方新闻稿把 1.6T 光 DSP 与 hyperscale/cloud optical network 部署相连，是 200G/lane 从模块样品走向平台化 portfolio 的重要供应链信号。来源：[Marvell 1.6T optical DSP platform](https://www.marvell.com/company/newsroom/marvell-1-6t-optical-dsp-ai-data-center-connectivity.html)
4. ⭐ **Eoptolink 公开 1.6T OSFP 产品页，覆盖 DR8、2xFR4、FRO/LRO 架构和 CMIS 5.0+ firmware。** 国内光模块厂商已经把 1.6T OSFP 作为 AI clusters/cloud data centers 的常规产品线展示，但实际出货、客户和价格仍需以后续公告或财报验证。来源：[Eoptolink 1.6T OSFP](https://www.eoptolink.com/)
5. 📌 **EDN 的 2026 CPO 状态文章把“AI scale-up 的电 I/O 瓶颈”与 packaging/服务性现实放在一起。** 该材料适合作为趋势参考：CPO 价值来自缩短电通道和降低 pJ/bit，但工程约束会落到 optical engine 可更换性、热路径、laser source 和良率。来源：[EDN CPO in 2026](https://www.edn.com/where-co-packaged-optics-cpo-technology-stands-in-2026/)

## 🖥️ AI服务器与超节点

1. 🔥 **NVIDIA DGX SuperPOD GB200 reference architecture 继续给出 NVL72 rack 内 NVLink Fabric 的明确结构。** 官方文档称一个 DGX GB200 rack 由 18 个 compute trays 和 9 个 NVLink switch trays 构成，同 rack 72 GPU full-mesh connectivity 依赖 NVL5；这类结构决定了 rack 内 copper/backplane、rack 间光互连和 leaf/spine 的边界。来源：[NVIDIA DGX SuperPOD network fabrics](https://docs.nvidia.com/dgx-superpod/reference-architecture-scalable-infrastructure-gb200/latest/network-fabrics.html)
2. 🔥 **Vera Rubin/NVL 路线的市场关注点已经从单卡性能转到 rack-scale 制造与 CPO readiness。** Tom's Hardware 7 月报道转述 Jensen Huang 称 Rubin roadmap intact，同时汇总了外部关于 Kyber NVL144、NVL576 与 CPO 技术挑战的传闻；这些内容不是官方规格，但提示超节点风险在 midplane、光 I/O 和系统装配。来源：[Tom's Hardware Vera Rubin report](https://www.tomshardware.com/tech-industry/artificial-intelligence/nvidias-huang-vows-to-deliver-giant-amounts-of-vera-rubin-company-says-that-our-roadmap-is-intact)
3. ⭐ **UALink 白皮书把开放 scale-up fabric 定位在 PCIe/CXL node-local 与 Ethernet/InfiniBand scale-out 之间。** 公开 candidate white paper 强调 purpose-built rack-scale fabric、multi-vendor competition、optical reach extension without protocol changes；对 NVLink 之外的 accelerator pod 生态有直接参考价值。来源：[UALink white paper PDF](https://ualinkconsortium.org/wp-content/uploads/2026/01/UALink_White_Paper_Publication_Candidate_FINAL_VERSION.pdf)
4. ⭐ **Synopsys 1.6T Ultra Ethernet IP solution 把 224G PHY、UEC MAC/PHY 和 IEEE 802.3dj/df 放进 scale-out AI cluster 语境。** 这说明 Ultra Ethernet 不只是交换机协议议题，还会影响 GPU/NIC/switch SoC 的 MAC/PCS/FEC 与 SerDes 集成。来源：[Synopsys 1.6T Ultra Ethernet IP](https://www.synopsys.com/designware-ip/interface-ip/ethernet/1-6t-ultra-ethernet-cs.html)

2. 🔥 **Pure DC 宣布芬兰 550MW campus，第一阶段 110MW 且 fully leased，并采用 Direct Liquid Cooling。** ITPro 报道称该项目总投资超过 75 亿欧元、可使用 700MVA renewable energy，代表 AI/HPC campus 选址正在把低碳电力、DLC 和余热利用作为标准要求。来源：[ITPro Pure DC Finland campus](https://www.itpro.com/infrastructure/data-centres/pure-dc-reveals-plans-for-550mw-data-center-campus-in-finland)

## 📄 论文速递

1. 🔥 **arXiv:2602.12521：Opus: Photonic Rail-Optimized Fabric in ML Datacenters。** 论文提出 photonic rail + OCS reconfiguration，用训练迭代中的通信阶段差异复用 optical circuit；适合评估 ML scale-out 网络的能耗/成本 trade-off。来源：[arXiv:2602.12521](https://arxiv.org/html/2602.12521v3)
2. 🔥 **arXiv:2603.21313：3D optoelectronics and co-packaged optics: when solving the wrong problems stalls deployment。** 论文提醒 CPO 不是单颗 optical engine 优化，而是 packaging、thermal、serviceability 和 system robustness 的系统架构承诺。来源：[arXiv:2603.21313](https://arxiv.org/abs/2603.21313)
3. 🔥 **arXiv:2606.23390：Symbol Rate-Code Rate Trade-offs for IM/DD 200G/400G per Lane LPO Transceivers。** 论文分析 PAM-4/6/8、FEC code rate 与带宽受限 IM/DD 系统，直接对应 200G/400G per-lane LPO 物理层权衡。来源：[arXiv:2606.23390](https://arxiv.org/pdf/2606.23390)
4. ⭐ **arXiv:2606.14052：Nonlinear Pluggable Optics。** 论文提出 DSP-free NLPO，用 photonic NGRC 补偿 PAM4 IM/DD 链路非线性和色散，目标是在 LPO 低功耗与 DSP robustness 之间找中间路线。来源：[arXiv:2606.14052](https://arxiv.org/html/2606.14052)
5. ⭐ **arXiv:2604.12566：Scalable 3D silicon nitride photonic interposer for high-density interconnects。** 论文面向 CPO/scale-up compute clusters，关注 3D SiN photonic interposer 如何处理 routing density、loss 和 packaging integration。来源：[arXiv:2604.12566](https://arxiv.org/html/2604.12566v1)
6. ⭐ **arXiv:2602.08284：2.5D co-packaged optical I/O chipsets on a SiON/Si interposer for 4 x 100G optical interconnection。** 论文实验展示 SiON/Si interposer、InP EML/PD 与 WDM routing 的 400G single-fiber optical I/O，对 2.5D co-packaged optical I/O 有参考价值。来源：[arXiv:2602.08284](https://arxiv.org/abs/2602.08284)
7. 📌 **arXiv:2603.28290：OptINC: Optical In-Network-Computing for Scalable Distributed Learning。** 论文把 optical devices 用于 gradient averaging/quantization，属于偏前瞻的 optical in-network computing，但能帮助理解通信瓶颈如何反推光互连架构。来源：[arXiv:2603.28290](https://arxiv.org/abs/2603.28290)
8. 📌 **arXiv:2605.26930：Phase-Efficient All-to-All Communication in Reconfigurable Networks。** 论文关注 ML/HPC all-to-all 在 reconfigurable optical networks 上的 phase efficiency，对 MoE 和 dense collective communication 有参考意义。来源：[arXiv:2605.26930](https://arxiv.org/html/2605.26930v2)
