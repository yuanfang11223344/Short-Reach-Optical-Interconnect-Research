## 🔌 SerDes与互连技术

1. 🔥 **OIF 448G 工作从 framework 进入 VSR/LR 接口定义。** Current Work 页面已经列出 CEI-448G-VSR 与 CEI-448G-LR：VSR 面向 chip-to-module，LR 面向 backplane，说明 448G 的讨论正在落到 channel reach、连接器、封装材料与 compliance 边界。[OIF](https://www.oiforum.com/technical-work/current-work/)
2. 🔥 **CEI-448G framework 把 112GHz 级 channel 能力和更高插损预算列为关键前提。** 对 SerDes 来说，448G 不是简单把 224G clock 加倍，而是封装、连接器、均衡、FEC 和功耗共同约束的问题。[OIF CEI-448G Framework PDF](https://www.oiforum.com/wp-content/uploads/OIF-FD-CEI-448G-01.0.pdf)
3. ⭐ **CEI-224G interoperability 仍是 448G 前的工程台阶。** OIF OFC 2025 demo 覆盖 112G/224G/448G 方向，224G-LR、MR 与 Linear 生态的互操作经验会直接影响 448G bring-up 与测试夹具设计。[OIF OFC 2025 Demo PDF](https://www.oiforum.com/wp-content/uploads/OIF_CEI_Demo_OFC2025.pdf)
4. ⭐ **Synopsys 把 224G SerDes 与 UALink/UEC 放在同一 AI/HPC 互连栈中解读。** 这类 IP 叙事表明 224G PHY 已经从单点高速接口进入 scale-up/scale-out protocol enablement 阶段。[Synopsys](https://www.synopsys.com/articles/224g-serdes-interoperability-ai-hpc.html)
5. 📌 **Socionext 的 DesignCon 2026 材料继续强调 224G/448G package realism。** 真实封装设计、仿真和量测数据对 SerDes 工程师比理想通道指标更有参考价值。[Socionext](https://socionextus.com/pressreleases/socionext-to-showcase-advanced-packaging-and-serdes-architecture-at-designcon-2026/)

## 🔦 光学与光模块

1. 🔥 **NVIDIA Spectrum-X Ethernet Photonics 把 CPO 与 200G SerDes 绑定到 Vera Rubin/Rubin 网络路线。** 官方材料称 CPO switch 面向更高网络能效、可靠性和 AI uptime，意味着光引擎不再只是 module 替代品，而是 rack-scale 网络平台的一部分。[NVIDIA Newsroom](https://nvidianews.nvidia.com/news/vera-rubin-full-production-agentic-ai-factory)
2. 🔥 **Spectrum-X 页面强调“把 optics 放到 switch ASIC 同 package”。** 这个表述确认 CPO 的验证边界包括 switch ASIC、silicon photonics、external laser、network OS 和运维可靠性，而不只是单颗 optical engine 指标。[NVIDIA Spectrum-X](https://www.nvidia.com/en-us/networking/spectrumx/)
3. ⭐ **CPO 产业判断分化：分析师看好长期，但短期 adoption 仍受 NPO、pluggable 和服务性约束。** MarketWatch 报道显示光器件股价和 CPO 预期已被市场交易，但工程落地仍要看 NVIDIA、Coherent、Lumentum 等平台节奏。[MarketWatch](https://www.marketwatch.com/story/missed-the-rally-in-optical-stocks-coherent-and-lumentum-just-got-more-attractive-according-to-j-p-morgan-ccee9f93)
4. ⭐ **硅光子在 800G/1.6T 模块中的占比继续成为产业主线。** C-Light 的市场观察称 800G 中 silicon photonics 份额已接近半数；该数据属于产业媒体口径，宜作为趋势参考而非官方统计。[C-Light](https://c-light.com/news/details/2026_Telecom_Optical_Module_Market.html)
5. 📌 **DesignCon 2026 现场报道提到 PCIe 7.0 over optics 与 800G LPO module demo。** 对短距互连来说，这类 demo 说明光链路正在从 Ethernet module 外溢到 PCIe/board-level I/O 探索。[Signal Integrity Journal](https://www.signalintegrityjournal.com/articles/4262-designcon-2026-ai-driven-infrastructure-and-system-level-design)

## 🖥️ AI服务器与超节点

1. 🔥 **UALink 2.0 将 In-Network Compute 纳入开放 scale-up fabric。** Common 2.0 把 reductions、aggregations、synchronization 和 collectives 推向 fabric 内部，可能改变 endpoint NIC/GPU 与 switch ASIC 的分工。[UALink 2.0 PR PDF](https://ualinkconsortium.org/wp-content/uploads/2026/04/UALink-2.0-Specification-PR_FINAL.pdf)
2. ⭐ **UALink Chiplet 1.01 面向 chiplet-based SoC 集成。** 规范说明其与 UCIe 3.0 对齐，开放 scale-up 互连开始覆盖 package 内/近 package 的接口、flow control 与管理问题。[UALink Specifications](https://ualinkconsortium.org/specification/)
3. 🔥 **Vera Rubin POD 继续强化“多 rack system 组成一个 AI supercomputer”的设计。** NVIDIA 把 GPU、CPU、NVLink、Spectrum-6 SPX、Quantum-X800/Spectrum-X 网络放在同一平台节奏中，说明超节点的发布门槛已经是整机柜/多机柜协同。[NVIDIA Developer Blog](https://developer.nvidia.com/blog/nvidia-vera-rubin-pod-seven-chips-five-rack-scale-systems-one-ai-supercomputer/)
4. ⭐ **UEC 1.0.2 仍是 scale-out Ethernet 的开放基线。** UEC 规范覆盖 transport、congestion、telemetry 与 compliance 方向，适合与 UALink scale-up 分层比较。[UEC Specification PDF](https://ultraethernet.org/wp-content/uploads/sites/20/2026/01/UE-Specification-1.0.2-1.pdf)

## 🏢 计算中心与基础设施

1. 🔥 **AI data center 供电架构继续向 facility-level DC 与 medium-voltage 方案演进。** arXiv:2606.25095 讨论 48V、LVDC、MVDC/SST 等路径，核心矛盾是转换级数、铜损、可靠性和 rack power density。[arXiv:2606.25095](https://arxiv.org/abs/2606.25095)
2. ⭐ **液冷交付重点从 cold plate 扩展到 CDU/chiller/控制链路。** Panasonic 的 400/800kW CDU 和 800/1200kW chiller 说明 AI 机房正在把冷却能力作为系统级基础设施交付，而不是单机柜附件。[Panasonic](https://news.panasonic.com/global/press/en260304-2)
3. ⭐ **AI load 与电网协同进入基础设施论文和政策项目。** 欧盟项目与 arXiv:2606.21064 都把数据中心负载灵活性、电网规划、能源效率放在同一问题空间。[European Commission](https://energy.ec.europa.eu/news/flagship-projects-ai-grids-and-data-centre-sustainability-2026-06-04_en) / [arXiv:2606.21064](https://arxiv.org/abs/2606.21064)
