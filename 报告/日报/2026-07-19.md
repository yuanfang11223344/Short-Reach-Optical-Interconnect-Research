## 🔌 SerDes与互连技术

1. 🔥 **OIF CEI-224G-MR、CEI-224G-Linear 仍是 1.6T 模块 host interface 与 LPO/LRO 架构选择的核心入口。** OIF current work 页面继续列出 CEI-224G-MR、CEI-224G-Linear、CEI-224G-LR 等项目；对 SerDes 设计来说，reach class、均衡能力、linear/retimed optical module 的边界会直接决定 DSP、retimer 和封装 channel budget。来源：[OIF Current Work](https://www.oiforum.com/technical-work/current-work/)
2. 🔥 **IEEE P802.3dj 继续承担 200G/lane、800GbE/1.6TbE 以太网规格收敛。** 该项目公开页面覆盖 200Gb/s per lane electrical/optical PHY、800GbE 与 1.6TbE；7 月 Montreal plenary 后，后续仍需跟踪 comment resolution、FEC、optical PMD 与 electrical baseline 的变化。来源：[IEEE P802.3dj Task Force](https://www.ieee802.org/3/dj/index.html)、[IEEE 802.3 interim sessions](https://www.ieee802.org/3/interims/index.html)
3. 🔥 **448G CEI 从“高速链路是否可测”走向系统可实现性讨论。** Ethernet Alliance/OIF 公开 panel 材料把 CEI-448G framework、3.2T 级信号生成采集、调制格式和测试挑战放在一起；工程重点是 400G/lane 可能如何在 DSP 复杂度、材料、封装损耗和功耗之间折中。来源：[Ethernet Alliance OIF 448G panel PDF](https://ethernetalliance.org/wp-content/uploads/2025/12/OIF_TEF_Panel2_2512.pdf)
4. ⭐ **Marvell 3nm 224G Long-Reach SerDes + co-packaged copper demo 强化 rack/tray 内短距铜互连路线。** Marvell 称其 224G LR SerDes 在 OFC 2026 与 Luxshare-Tech 演示中驱动 2.5 m CPC-backplane-CPC 通道；这类方案适合观察 AI tray、rack 内 copper reach 与光互连边界。来源：[Marvell 224G Long-Range SerDes](https://www.marvell.com/blogs/224g-long-range-serdes-scale-up-scale-inside.html)
5. ⭐ **Synopsys 与 Cadence 的 224G SerDes IP 页面都把 AI/HPC、1.6T Ethernet 和 UALink/UEC 语境写进产品定位。** 对芯片团队来说，224G PHY 已经不是单独模拟 IP，而是与 MAC/PCS/FEC、验证 IP、封装建模和系统通道协同采购的接口平台。来源：[Synopsys 224G Ethernet PHY IP](https://www.synopsys.com/designware-ip/interface-ip/ethernet/224g-ethernet-phy-ip.html)、[Cadence 224G-LR SerDes PHY](https://www.cadence.com/en_US/home/tools/silicon-solutions/design-ip/high-speed-ethernet/224g-lr-ser-des-phy.html)

## 🔦 光学与光模块

1. 🔥 **OIF OFC 2026 multi-vendor demo 把 CEI-224G、CEI-448G、CMIS、co-packaging 与 EEI 放进同一互操作框架。** 这说明下一代 AI 光互连不只是 1.6T/3.2T 模块速率升级，而是电接口、模块管理、线性光、CPO 与节能接口共同收敛。来源：[OIF at OFC 2026](https://www.oiforum.com/meetings-events/oif-ofc-2026/)
2. 🔥 **OFC 2026 仍是回查 1.6T/3.2T、CPO、silicon photonics 与 optical I/O 论文/演示的主索引。** Optica 官方会议入口与产业总结都把 AI data center optical connectivity 作为主线；后续日报应优先跟踪可以落到 200G/lane、400G/lane、packaging 和模块管理的证据。来源：[Optica OFC 2026](https://opg.optica.org/conference.cfm?meetingid=5)、[OFC 2026 Outlook](https://tspasemiconductor.substack.com/p/ofc-2026-outlook-ai-data-center-optical)
3. ⭐ **Marvell 1.6T optical DSP portfolio 将 Ara T、Ara X、Petra 分别指向 TRO、可靠性增强与 gearbox。** 官方新闻稿把这些 DSP 与 hyperscale/cloud optical network 部署相连，表明 1.6T 从单点芯片走向面向不同模块架构的组合产品线。来源：[Marvell 1.6T optical DSP platform](https://www.marvell.com/company/newsroom/marvell-1-6t-optical-dsp-ai-data-center-connectivity.html)、[Marvell optical DSP](https://www.marvell.com/solutions/data-center/optical-dsp.html)
4. ⭐ **Eoptolink 公开 1.6T OSFP DR8、2xFR4、FRO/LRO 等产品信息，国内光模块厂商已将 1.6T 作为 AI cluster 常规展示线。** 公开产品页可用于确认形态和应用方向，实际出货、客户与价格仍需财报或公告验证。来源：[Eoptolink 1.6T OSFP](https://www.eoptolink.com/)
5. 📌 **EDN 2026 CPO 状态文章强调 deployment 阻力来自 packaging、热、可维护性和 laser source，不只是 pJ/bit。** 该文适合作为趋势参考：CPO 的价值在于缩短电通道、降低能耗和提升带宽密度，但系统团队要提前处理光引擎更换、良率与冷却路径。来源：[EDN CPO in 2026](https://www.edn.com/where-co-packaged-optics-cpo-technology-stands-in-2026/)

## 🖥️ AI服务器与超节点

1. 🔥 **NVIDIA DGX SuperPOD GB200 reference architecture 明确给出 NVL72 rack 内 NVLink Fabric 结构。** 官方文档称一个 DGX GB200 rack 由 18 个 compute trays 与 9 个 NVLink switch trays 构成，同 rack 72 GPU 互连依赖 NVL5；这会直接决定 tray/backplane、rack 内铜缆和 rack 间光互连的分界。来源：[NVIDIA DGX SuperPOD network fabrics](https://docs.nvidia.com/dgx-superpod/reference-architecture-scalable-infrastructure-gb200/latest/network-fabrics.html)
2. 🔥 **Rubin/NVL 路线的风险点正在从单卡性能扩展到 rack-scale 制造、midplane 与 CPO readiness。** Tom's Hardware 7 月报道转述 Jensen Huang 称 Rubin roadmap intact，同时汇总外部关于 Kyber NVL144/NVL576 和 CPO 的传闻；这些不是官方规格，但提示超节点交付瓶颈可能在系统装配和光 I/O。来源：[Tom's Hardware Vera Rubin report](https://www.tomshardware.com/tech-industry/artificial-intelligence/nvidias-huang-vows-to-deliver-giant-amounts-of-vera-rubin-company-says-that-our-roadmap-is-intact)
3. ⭐ **UALink candidate white paper 将开放 scale-up fabric 定位在 PCIe/CXL node-local 与 Ethernet/InfiniBand scale-out 之间。** 白皮书强调 rack-scale、multi-vendor competition、without-protocol-change optical reach extension；这对 NVLink 之外的 accelerator pod 互连生态有直接参考价值。来源：[UALink white paper PDF](https://ualinkconsortium.org/wp-content/uploads/2026/01/UALink_White_Paper_Publication_Candidate_FINAL_VERSION.pdf)
4. ⭐ **Synopsys 1.6T Ultra Ethernet IP solution 把 224G PHY、UEC MAC/PHY 与 IEEE 802.3dj/df 放到 scale-out AI cluster 场景。** 这说明 Ultra Ethernet 会落实到 GPU/NIC/switch SoC 的 MAC、PCS、FEC 和 SerDes 集成，而不只是网络协议层讨论。来源：[Synopsys 1.6T Ultra Ethernet IP](https://www.synopsys.com/designware-ip/interface-ip/ethernet/1-6t-ultra-ethernet-cs.html)

## 📄 论文速递

1. 🔥 **arXiv:2602.12521：Opus: Photonic Rail-Optimized Fabric in ML Datacenters。** 论文提出 photonic rail + OCS reconfiguration，用训练迭代中的通信阶段差异复用 optical circuit；适合评估 ML scale-out 网络的能耗/成本 trade-off。来源：[arXiv:2602.12521](https://arxiv.org/html/2602.12521v3)
2. 🔥 **arXiv:2603.21313：3D optoelectronics and co-packaged optics: when solving the wrong problems stalls deployment。** 论文提醒 CPO 不是单颗 optical engine 优化，而是 packaging、thermal、serviceability 和 system robustness 的系统架构承诺。来源：[arXiv:2603.21313](https://arxiv.org/abs/2603.21313)
3. 🔥 **arXiv:2606.23390：Symbol Rate-Code Rate Trade-offs for IM/DD 200G/400G per Lane LPO Transceivers。** 论文分析 PAM-4/6/8、FEC code rate 与带宽受限 IM/DD 系统，直接对应 200G/400G per-lane LPO 物理层权衡。来源：[arXiv:2606.23390](https://arxiv.org/pdf/2606.23390)
4. ⭐ **arXiv:2606.14052：Nonlinear Pluggable Optics。** 论文提出 DSP-free NLPO，用 photonic NGRC 补偿 PAM4 IM/DD 链路非线性和色散，目标是在 LPO 低功耗与 DSP robustness 之间找中间路线。来源：[arXiv:2606.14052](https://arxiv.org/html/2606.14052)
5. ⭐ **arXiv:2604.12566：Scalable 3D silicon nitride photonic interposer for high-density interconnects。** 论文面向 CPO/scale-up compute clusters，关注 3D SiN photonic interposer 如何处理 routing density、loss 和 packaging integration。来源：[arXiv:2604.12566](https://arxiv.org/html/2604.12566v1)
6. ⭐ **arXiv:2602.08284：2.5D co-packaged optical I/O chipsets on a SiON/Si interposer for 4 x 100G optical interconnection。** 论文展示 SiON/Si interposer、InP EML/PD 与 WDM routing 的 400G single-fiber optical I/O，对 2.5D co-packaged optical I/O 有参考价值。来源：[arXiv:2602.08284](https://arxiv.org/abs/2602.08284)
7. 📌 **arXiv:2603.28290：OptINC: Optical In-Network-Computing for Scalable Distributed Learning。** 论文把 optical devices 用于 gradient averaging/quantization，属于偏前瞻的 optical in-network computing，但有助于理解通信瓶颈如何反推光互连架构。来源：[arXiv:2603.28290](https://arxiv.org/abs/2603.28290)
8. 📌 **arXiv:2605.26930：Phase-Efficient All-to-All Communication in Reconfigurable Networks。** 论文关注 ML/HPC all-to-all 在 reconfigurable optical networks 上的 phase efficiency，对 MoE 和 dense collective communication 有参考意义。来源：[arXiv:2605.26930](https://arxiv.org/html/2605.26930v2)
