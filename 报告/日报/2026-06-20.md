## 🔌 SerDes与互连技术

1. 🔥 **224G retimer 已经被明确当作 UALink、ESUN、Ethernet 的共用 Layer 1 底座**
   - 要点：Credo Blue Heron 224G AI scale-up retimer 官方写明同时支持 UALink、ESUN、Ethernet，并面向 extended cable 与 PCB backplane links。
   - 工程意义：开放 scale-up 协议之争，首先落到 224G lane 的恢复能力、协议共存和 rack-scale cable backplane 可实现性。
   - 来源：https://investors.credosemi.com/news-events/news/news-details/2026/Credo-Introduces-Industrys-First-224G-Multiprotocol-AI-Scale-Up-Retimer-Supporting-UALink-ESUN-and-Ethernet/default.aspx

2. 🔥 **Credo 公开把 40+dB 链路恢复、3nm、30-Tap FFE + 16-Tap 反射消除写成卖点**
   - 要点：Blue Heron 宣称可完整恢复 40+dB 的 224G 链路，并采用 3nm 工艺与 30-Tap FFE、16-Tap reflection cancellation。
   - 工程意义：这类指标直接对应从 half-rack 走向整柜 cable/backplane 的 SI 裕量，而不是单纯的 PHY 代际升级。
   - 来源：https://investors.credosemi.com/news-events/news/news-details/2026/Credo-Introduces-Industrys-First-224G-Multiprotocol-AI-Scale-Up-Retimer-Supporting-UALink-ESUN-and-Ethernet/default.aspx

3. ⭐ **OIF 正把 CEI-224G-Linear 与 Compute Optics Interface 同时推进**
   - 要点：OIF current work 页面显示，CEI-224G-Linear 面向 next-gen Ethernet、UEC、AI/ML 的 224G full linear optical modules；COI 面向 PCIe、NVLink、UALink 的低时延光接口。
   - 工程意义：LPO、linear optics、compute optics 不再只是厂商路线图，而是进入跨生态的接口定义阶段。
   - 来源：https://www.oiforum.com/technical-work/current-work/

4. ⭐ **UALink 1.0 仍然是开放 scale-up 的最清晰系统上限定义之一**
   - 要点：UALink 规格页写明，UALink 200G 1.0 支持每 lane 200G，可覆盖一个 AI computing pod 内最多 1,024 accelerators。
   - 工程意义：这为 retimer、switch、cable、AEC 与 package I/O 预算给出了更现实的系统边界。
   - 来源：https://ualinkconsortium.org/specification/

5. 📌 **UALink 2.0 已经在 2026-04-07 发布，开始把 in-network compute、chiplet、manageability 拉进规范**
   - 要点：UALink 新闻页显示 2026-04-07 发布了四份规格，配套新闻稿摘要提到新增 in-network compute、chiplet definition、manageability。
   - 工程意义：开放 AI 互连正在从“先把链路跑起来”进入“如何做系统功能扩展”的阶段。
   - 来源：https://ualinkconsortium.org/news/
   - 来源：https://ualinkconsortium.org/wp-content/uploads/2026/04/UALink-2.0-Specification-PR_FINAL.pdf

6. 📌 **ESUN 1.0 的 4-byte header 说明 Ethernet scale-up 正在认真优化小包效率**
   - 要点：OCP 对 ESUN 1.0 的介绍明确写到，用 4-byte ESUN Header 替换传统 28-48 byte 的 IP/UDP header stack，以降低小消息开销。
   - 工程意义：这直指 collective 和同步类流量最敏感的 payload efficiency 问题，和纯粹追求链路速率不是一回事。
   - 来源：https://www.opencompute.org/blog/the-ocp-esun-10-specification-has-been-released

## 🔦 光学与光模块

1. 🔥 **Broadcom 把 400G/lane optical DSP、400G EML/PD、1.6T 与 3.2T 路线一次性串起来**
   - 要点：Broadcom 在 OFC 2026 公告中写明，Taurus 是业界首个 400G/lane optical DSP，并与 400G EML/PD 配套，面向低功耗 1.6T，同时为 3.2T 铺路。
   - 工程意义：400G/lane 正在从研究议题转为 DSP + optics 器件栈的联合推进，直接影响 204.8T 平台的光接口代际。
   - 来源：https://investors.broadcom.com/news-releases/news-release-details/broadcom-showcases-industry-leading-solutions-scaling-ai

2. 🔥 **Eoptolink 把 1.6T DR4 OSFP 明确做成 8×200G 电到 4×400G 光的桥接模块**
   - 要点：Eoptolink 3 月公告称，其 1.6T DR4 OSFP 使用 8×200G electrical interface 对接 4×400G optical interface，并可接入当前 102.4T switches。
   - 工程意义：这是从 200G/lane host 侧过渡到 400G/lambda 光侧的一种现实迁移路径。
   - 来源：https://eoptolink.com/news/13-new-products/365-eoptolink-demos-imdd-400g-per-lambda-based-1-6t-dr4-optical-transceiver-solution-at-ofc-2026

3. 🔥 **同一方案把 1.6T 相比 DR8 的所需纤芯数降低 50%**
   - 要点：Eoptolink 官方明确写到，该 1.6T DR4 相比上一代 1.6T DR8 架构可减少 50% 的 required fiber count。
   - 工程意义：对前面板密度、配线复杂度和大规模 AI 集群运维，这比纸面带宽翻倍更关键。
   - 来源：https://eoptolink.com/news/13-new-products/365-eoptolink-demos-imdd-400g-per-lambda-based-1-6t-dr4-optical-transceiver-solution-at-ofc-2026

4. ⭐ **Eoptolink 已经把 6.4T NPO 和 12.8T XPO 一起公开摆上 OFC 2026 展台**
   - 要点：其 6.4T NPO 模块为 32 lanes、每 lane 200Gbps；官方同时列出 12.8T XPO、400G/lambda 1.6T DR4、200G/lambda 1.6T FRO/LRO/LPO 系列展示。
   - 工程意义：国内厂商不只是跟进 1.6T pluggable，而是在提前卡位 NPO/XPO 封装密度路线。
   - 来源：https://www.eoptolink.com/news/13-new-products/366-eoptolink-unveils-high-density-6-4t-npo-solution-at-ofc-2026

5. ⭐ **Coherent 的 1.6T-DR8 OSFP 产品页已经把 IEEE P802.3dj 与 OIF CEI-224G-LR 写进 host/line interface**
   - 要点：Coherent 当前产品页显示，该模块为 8×200G PAM4，兼容 IEEE P802.3dj 和 OIF CEI-224G-LR host/line interfaces。
   - 工程意义：这类正式产品页比展会口径更接近量产交付接口组合。
   - 来源：https://www.coherent.com/networking/transceivers/datacom/FTCF2519E3PCA

6. 📌 **224G 测试生态已经从实验室演示走向 1.6T 光网络验证**
   - 要点：Keysight 2026-03-13 发布的 224G 测试方案覆盖 end-to-end electrical 与 optical validation，直指 1.6T optical network validation。
   - 工程意义：224G 的系统瓶颈开始落到一致性验证、参考接收机建模和量产测试，而不是“有没有器件”。
   - 来源：https://www.keysight.com/us/en/about/newsroom/news-releases/2026/0313-pr26-049-keysight-introduces-new-224g-test-solutions-to-enable-1-6t-optical-network-validation.html

## 📄 论文速递

1. 🔥 **arXiv:2602.12521 / Photonic Rails in ML Datacenters with Opus**
   - 一句话：论文提出用可重构 optical circuit switches 替换 rail 中的 electrical packet switches，并在 2,048 GPUs 级仿真中评估 power/cost 与训练开销权衡。
   - 为什么值得看：很贴近 AI scale-out rail fabric 的系统级光互连问题，而不是单器件优化。
   - 链接：https://arxiv.org/html/2602.12521v2

2. 🔥 **arXiv:2602.10505 / Scaling Routers with In-Package Optics and High-Bandwidth Memories**
   - 一句话：论文把 HBM、chiplets 和 in-package optics 引入 router 架构，目标是在单封装内实现 petabit/sec 级路由能力，并明确把功耗视为扩展主瓶颈。
   - 为什么值得看：对 future switch/router 封装路线和 CPO/NPO 讨论有直接启发。
   - 链接：https://arxiv.org/abs/2602.10505

3. ⭐ **arXiv:2606.19674 / Design Considerations for Phase Modulation in Testable Photonic Systems and Co-packaged Optics**
   - 一句话：6 月 18 日论文比较 thermal 与 carrier-based phase modulation，聚焦 testable photonic systems 与 CPO 校准/测试需求。
   - 为什么值得看：CPO 真正走向量产时，测试与校准机制会和器件本身同等重要。
   - 链接：https://arxiv.org/html/2606.19674v1

4. ⭐ **arXiv:2605.28971 / Micro-Transfer Printing of Lithium Niobate on 200 mm Silicon Photonics**
   - 一句话：6 月 15 日版本讨论把 lithium niobate 集成到 200 mm silicon photonics 平台，直接对应高速、低能耗调制器的制造路径。
   - 为什么值得看：这是 TFLN 与 SiPho 工艺融合走向可制造化的重要方向。
   - 链接：https://arxiv.org/html/2605.28971v2
