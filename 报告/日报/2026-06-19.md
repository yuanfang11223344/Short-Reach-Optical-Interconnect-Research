## 🔌 SerDes与互连技术

1. 🔥 **OIF 持续推进 CEI-224G-Linear，明确把 linear optics 指向 AI/ML 与 UEC**
   - 要点：OIF current work 页面仍把 CEI-224G-Linear 列为当前项目，目标是支持 224G full linear optical modules，应用明确覆盖 Ethernet、UEC、AI/ML。
   - 工程含义：224G linear 已不只是单点器件路线，而是在标准层给 LPO、NPO、CPO 继续预留接口空间。
   - 来源：https://www.oiforum.com/technical-work/current-work/

2. 🔥 **OCP ESUN 1.0 已发布，开放 Ethernet 正在正式进入 scale-up AI 网络**
   - 要点：OCP 3 月发布 ESUN 1.0，重点包括 lossless connectivity、link level reliability 和面向 scale-up 网络的 framing/transport 能力。
   - 工程含义：这会直接影响 AI 机内/机间 fabric 对交换芯片、retimer、NIC 和软件栈的边界划分。
   - 来源：https://www.opencompute.org/blog/the-ocp-esun-10-specification-has-been-released

3. ⭐ **UALink 2.0 已上线下载页，引入 In-Network Compute**
   - 要点：UALink 官网当前提供 Common 2.0 specification，说明新增 In-Network Compute，目标是改善分布式训练和推理中的扩展效率。
   - 工程含义：开放 scale-up 协议正在向“传输 + 交换内计算 + 管理”一体化演进，对 200G/224G PHY 与 switch 生态是持续催化。
   - 来源：https://ualinkconsortium.org/specification/

4. ⭐ **OIF 在 OFC 2026 同时展示 CEI-224G 与 CEI-448G，多厂互通开始为 3.2T 铺路**
   - 要点：OIF OFC 2026 页面显示，现场 demo 已覆盖 CEI-224G、CEI-448G 以及 EEI/co-packaging。
   - 工程含义：448G 仍处于 pathfinding 阶段，但生态把它与 224G 互通展示放到同一舞台，说明 3.2T 前期定义已被提前拉动。
   - 来源：https://www.oiforum.com/oif-demonstrates-industry-wide-interoperability-at-scale-at-ofc-2026-advancing-energy-efficiency-performance-and-capacity-for-ai-era-data-center-networks/

5. 📌 **Synopsys 把 224G PHY 直接对齐 IEEE/OIF，并明确支持 UALink 200G**
   - 要点：Synopsys 224G Ethernet PHY IP 页面强调其满足 IEEE 802.3 与 OIF 电规范，并支持 UALink 200G。
   - 工程含义：说明 224G PHY 正在收敛成 AI fabric 与 Ethernet transport 的共用物理层能力。
   - 来源：https://www.synopsys.com/designware-ip/interface-ip/ethernet/224g-ethernet-phy-ip.html

6. 📌 **Cadence 将 224G LR SerDes 定位为 1.6T/800G AI 基础设施底座**
   - 要点：Cadence 页面明确称其 224G LR SerDes PHY 支持 LR/MR/VSR，面向 hyperscale data center 与 AI infrastructures。
   - 工程含义：单一 224G PAM4 PHY 同时覆盖 scale-up 和 scale-out，正成为主流叙事。
   - 来源：https://www.cadence.com/en_US/home/tools/silicon-solutions/design-ip/high-speed-ethernet/224g-lr-ser-des-phy.html

## 🔦 光学与光模块

1. 🔥 **Broadcom 推出 400G/lane optical DSP，把 1.6T 与 3.2T 路线连到一起**
   - 要点：Broadcom 3 月发布 Taurus，官方称其为首款 400G/lane optical DSP，面向 1.6T transceiver，并为 204.8T/3.2T 网络打基础。
   - 工程含义：400G/lane 从器件预研进入平台化竞争，EML/PD、测试、封装和热设计都将被同步拉高。
   - 来源：https://investors.broadcom.com/news-releases/news-release-details/broadcom-delivers-industrys-first-400glane-optical-dsp-next

2. 🔥 **Marvell 扩展 1.6T optical DSP 平台，继续强化 end-to-end connectivity 叙事**
   - 要点：Marvell 3 月宣布扩展 1.6T optical DSP 组合，并把 advanced SerDes、DSP 与 broader connectivity portfolio 绑定到 AI data center。
   - 工程含义：1.6T 周期的竞争核心正在从单颗 DSP 走向“交换芯片 + DSP + optics + system”的整链协同。
   - 来源：https://investor.marvell.com/news-events/press-releases/detail/1013/marvell-ushers-in-the-1-6t-era-with-expanded-optical-dsp-platform-portfolio-redefining-ai-data-center-end-to-end-connectivity

3. 🔥 **Credo 完成 DustPhotonics 收购，SiPho PIC 正式并入其光互连栈**
   - 要点：Credo 6 月完成 DustPhotonics 收购，官方说明其 silicon photonics PIC 能力将覆盖 800G、1.6T、3.2T 的 NPO/CPO 组合。
   - 工程含义：中型互连厂商也在向“SerDes/DSP/SiPho/系统集成”垂直整合，NPO/CPO 不再只是少数巨头游戏。
   - 来源：https://investors.credosemi.com/news-events/news/news-details/2026/Credo-Completes-Acquisition-of-DustPhotonics/default.aspx

4. ⭐ **Eoptolink 展示 400G per lambda 的 1.6T DR4 OSFP**

   - 工程含义：400G/lambda 不再只停留在头部欧美器件商演示，国内模块链条也开始把 1.6T DR4 推到公开样机层。
   - 来源：https://eoptolink.com/news/13-new-products/365-eoptolink-demos-imdd-400g-per-lambda-based-1-6t-dr4-optical-transceiver-solution-at-ofc-2026

   - 要点：其 OFC 2026 新闻稿称 6.4T NPO 方案基于 SiPh，32 lanes x 200Gbps，面向 AI data center interconnect density challenge。
   - 工程含义：国内厂商已不只跟随 800G/1.6T pluggable，也开始公开卡位 NPO 密度路线。
   - 来源：https://www.eoptolink.com/news/13-new-products/366-eoptolink-unveils-high-density-6-4t-npo-solution-at-ofc-2026
