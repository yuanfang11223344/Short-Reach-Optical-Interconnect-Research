# 芯片互连每日资讯 2026-06-30

> 生成时间：2026-06-30（Asia/Shanghai）。本期按 6 个方向检索；当天新增不足的方向采用近期一手资料做状态复核，并明确区分论文新提交、官方近期进展与持续跟踪项。

## 🔌 SerDes与互连技术

1. 🔥 **OIF 的 448G electrical interface 已从讨论框架进入 VSR/LR 项目化阶段**
   - CEI-448G-VSR/LR 与既有 CEI-224G-LR 并列进入工作列表，应用覆盖 chip-to-module、chip-to-chip 与 chassis 内/间连接。
   - 技术视角：448G/lane 不只是 baud rate 翻倍，ADC/DAC、clocking、FEC、reference package、channel model 与 compliance test 必须联合收敛。
   - 来源：https://www.oiforum.com/technical-work/current-work/

2. 🔥 **IEEE P802.3dj Draft 3.1 的 comment database 成为 200G/lane 最直接的工程检查表**
   - SA ballot comments 覆盖 200G/lane electrical/optical PMD、FEC、管理与测试条款；设计团队应把 comment resolution 与自身 channel/package 假设逐项映射。
   - 来源：https://www.ieee802.org/3/dj/comments/index.html

3. ⭐ **224G retimer 正从单一 Ethernet 器件演化为多协议 scale-up PHY 平台**
   - Credo Blue Heron 官方资料同时列出 UALink、ESUN 与 Ethernet，说明 lane repair、BER telemetry、latency 和 RAS 的实现将逐步平台化。
   - 来源：https://investors.credosemi.com/news-events/news/news-details/2026/Credo-Introduces-Industrys-First-224G-Multiprotocol-AI-Scale-Up-Retimer-Supporting-UALink-ESUN-and-Ethernet/default.aspx

4. ⭐ **1.6T optical DSP 已把 224G/lane、retimed optics 与 linear receive 纳入同一产品族**
   - Credo Cardinal 采用 3 nm、224G/lane 架构；官方定位同时覆盖 retimed 1.6T optics 与 linear-receive implementation。
   - 来源：https://investors.credosemi.com/news-events/news/news-details/2026/Credo-Introduces-Cardinal-A-LowPower-1-6T-Optical-DSP-Family-Engineered-for-MassiveScale-AI-Fabrics/default.aspx

## 🔦 光学与光模块

1. 🔥 **新论文把 CPO-enabled silicon photonics 的硬件可信问题提到设计层**
   - arXiv:2606.27612 研究利用 silicon photonics hardware fingerprinting 增强 co-packaged optics 的 security assurance。
   - 技术视角：CPO 进入量产后，光电 die 来源、封装替换与器件级认证将与链路性能同样成为系统验收项。
   - 来源：https://arxiv.org/abs/2606.27612

2. 🔥 **OpenLight 的 3.2T DR8 PIC 以 400G/lane EAM 指向下一代 pluggable/CPO/NPO**
   - 官方资料显示 3.2T DR8 prototype PIC 已提供样片，同时提供 1.6T DR8 LRO/LPO variants；其异质集成路线把 InP laser/EAM 与 silicon photonics 合并到 PIC。
   - 来源：https://www.ofcconference.org/news-media/exhibitor-news/openlight-introduces-3-2t-dr8-silicon-photonics-pic-s-as-well-as-1-6t-dr8-lro-and-lpo-variants/

3. ⭐ **Coherent 用 socketed CPO、VCSEL CPO 和 400G InP modulator 并行验证多条技术路线**
   - 6.4T（32×200G）socketed CPO 配套 ELS，说明早期 CPO 会优先解决可维护性、laser serviceability 与封装良率，而不会只追求最高集成度。
   - 来源：https://www.coherent.com/news/press-releases/coherent-co-packaged-optics-cpo-technologies-ofc-2026

4. ⭐ **NLM Photonics 开始提供 1.6T/3.2T silicon-organic hybrid PIC 样片**
   - 官方 OFC 资料将 1.6T DR8 与 3.2T SOH PIC 定位于低 drive-voltage、高带宽光引擎候选；实际量产仍需关注 organic material 的温度寿命、封装与 foundry repeatability。
   - 来源：https://www.ofcconference.org/news-media/exhibitor-news/nlm-photonics-initiates-sampling-of-1-6t-and-3-2t-silicon-organic-hybrid-pics/

## 🖥️ AI服务器与超节点

1. 🔥 **Vera Rubin NVL72 把 scale-up、scale-out、storage 与 rack power/cooling 合并为平台问题**
   - NVIDIA 官方架构由 Vera CPU、Rubin GPU、NVLink 6 Switch、ConnectX-9、BlueField-4 与 Spectrum-6/Quantum-X800 组成，系统边界已经超出单个 compute tray。
   - 技术视角：验证重点需从 SerDes BER 扩展到 collective latency、failure domain、checkpoint bandwidth 与热瞬态。
   - 来源：https://nvidianews.nvidia.com/news/nvidia-vera-rubin-platform

2. ⭐ **GB300 NVL72 dual-plane reference architecture 给出 8-rack、576-GPU 的网络落地样例**
   - NVIDIA 参考架构把 Spectrum-X compute fabric 与管理/存储网络分层，可作为 leaf/spine oversubscription、cabling 与 failure isolation 的具体设计输入。
   - 来源：https://docs.nvidia.com/enterprise-reference-architectures/nvl72-ai-factory-with-gb300-nvl72-dual-plane-networking-architecture.pdf

3. ⭐ **多协议 224G PHY 暗示 UALink、ESUN、Ethernet 的差异将更多上移到协议和软件层**
   - 同一 retimer 支持多个 scale-up/scale-out 协议后，竞争焦点会转向 memory semantics、ordering、congestion control、telemetry 与 orchestration。
   - 来源：https://investors.credosemi.com/news-events/news/news-details/2026/Credo-Introduces-Industrys-First-224G-Multiprotocol-AI-Scale-Up-Retimer-Supporting-UALink-ESUN-and-Ethernet/default.aspx

## 🏢 计算中心与基础设施

1. 🔥 **800VDC 与 liquid cooling 正被作为同一 rack/facility co-design 问题讨论**
   - Schneider Electric 指出早期 800VDC power sidecar 可接入既有设施，但规模扩大后会推动集中式 800VDC、MV-to-DC conversion 及 power electronics cooling。
   - 来源：https://blog.se.com/datacenter/2026/04/15/800-vdc-data-center-cooling-infrastructure-evolution-operational-impact/

2. ⭐ **AI factory cooling 的设计边界扩展到 facility heat-rejection system**
   - 近期官方技术文章把 cold plate、CDU/TCS 与 chiller/free-cooling 视为完整热链；仅优化 rack 内温度不足以判断集群持续性能与 water/energy cost。
   - 来源：https://blog.se.com/datacenter/2026/06/12/ai-factory-cooling-vs-cloud-data-centers-why-liquid-cooling-is-essential-for-high-density-ai-workloads/

3. ⭐ **GaN 在 AI data center 供电链中的价值是 stage-specific，而非全链替代**
   - arXiv:2606.25281 比较 PFC、isolated DC/DC、48V intermediate bus 与 point-of-load 阶段，强调 topology、package、EMI、thermal 与 reliability 共同决定收益。
   - 来源：https://arxiv.org/abs/2606.25281
