## 🔌 SerDes与互连技术

1. 🔥 **IEEE P802.3dj 继续定义 200G/lane 到 1.6TbE 的主干标准。** 官方 task force 页面列出 200 Gb/s、400 Gb/s、800 Gb/s、1.6 Tb/s Ethernet 项目文档、draft comment 和 adopted timeline；对 224G PAM4 SerDes、800G/1.6T 模组和交换 ASIC 端口规划仍是第一优先级标准来源。来源：[IEEE P802.3dj](https://www.ieee802.org/3/dj/index.html)
2. 🔥 **OIF CEI-224G-MR/LR 项目把 chip-to-chip、backplane/copper cable reach 目标正式拆开。** OIF current work 页面显示 CEI-224G-MR 目标至少 500 mm、最多一个连接器，CEI-224G-LR 目标至少 1000 mm、最多两个连接器；这直接影响 1.6T 端口附近 retimer/DSP 放置与 advanced materials 选型。来源：[OIF Current Work](https://www.oiforum.com/technical-work/current-work/)
3. 🔥 **448G CEI 已进入“框架与可达性”讨论阶段。** Ethernet Alliance/OIF 448G 材料强调 448G 在达到 224G 类似 reach 时会面临更高 Nyquist 频率、铜互连带宽和功耗约束；对 3.2T 模块电接口来说，调制、封装和 equalization 方案仍需要共同收敛。来源：[Ethernet Alliance OIF 448G panel PDF](https://ethernetalliance.org/wp-content/uploads/2025/12/OIF_TEF_Panel2_2512.pdf)
4. ⭐ **Broadcom 把 224G PAM4 与 AI cluster 的 1.6T interface、数百 lane per device 直接关联。** 其 SerDes for AI infrastructure 页面强调从 rack 到 data center 的 bandwidth density 与 low-latency link，是交换芯片、光 DSP 和 SerDes IP 协同设计的背景材料。来源：[Broadcom SerDes for AI Infrastructure](https://www.broadcom.com/topics/what-is-serdes-for-ai-infrastructure)
5. ⭐ **Synopsys 224G SerDes 文章把 UEC/UALink 与 224G PHY 放在同一系统语境。** 文章指出 UEC 面向 scale-out、UALink 面向 scale-up，两者都依赖新一代 224G SerDes；对 SoC 集成而言，重点是协议、equalization、封装、compliance test 一起验证。来源：[Synopsys 224G SerDes interoperability](https://www.synopsys.com/articles/224g-serdes-interoperability-ai-hpc.html)

3. ⭐ **Marvell 称 1.6T optical DSP 平台正在量产支持 hyperscale/cloud 部署。** 官方新闻稿称 Ara 3nm 1.6T platform 已面向全球客户 mass volume shipping，并扩展 1.6T optical DSP portfolio；这是 200G/lane 模块从工程样机进入部署节奏的重要供应链信号。来源：[Marvell 1.6T optical DSP platform](https://www.marvell.com/company/newsroom/marvell-1-6t-optical-dsp-ai-data-center-connectivity.html)
4. ⭐ **Marvell 1.6T LPO chipset 仍是 short-reach 去 DSP 路线的关键参考。** 该 LPO TIA 与 laser driver chipset 面向 800G/1.6T linear-drive pluggable optics，价值在于降低模块端 DSP 功耗，但系统层面需要 host SerDes、链路预算和互操作共同兜底。来源：[Marvell 1.6T LPO chipset](https://www.marvell.com/company/newsroom/marvell-introduces-1-6-tbps-lpo-chipset.html)
5. 📌 **QSFP-DD1600/OSFP224 路线正在把 200G SerDes 与 1.6T form factor 绑定。** Cisco Live 光模块资料显示 QSFP-DD 从 400G/800G 演进到 1.6T，200G SerDes 成为下一代 pluggable optics 的关键接口；适合作为 form factor 与后向兼容讨论材料。来源：[Cisco Live 400G/800G/Terabit Pluggable Optics PDF](https://www.ciscolive.com/c/dam/r/ciscolive/global-event/docs/2025/pdf/BRKOPT-2699.pdf)

## 🖥️ AI服务器与超节点

1. 🔥 **NVIDIA NVL72 AI Factory 参考架构给出 rack-scale networking 的明确端口预算。** 官方 network logical architecture 页面列出每个 scalable unit 的 GPU compute traffic、CPU fabric、support server 和 customer network 连接数量；这说明 GPU rack 不再只是服务器数量问题，而是 400G 上行、leaf/spine 和 software requirement 的共同约束。来源：[NVIDIA NVL72 network logical architecture](https://docs.nvidia.com/enterprise-reference-architectures/nvl72-ai-factory/latest/network-logical-architecture.html)
2. 🔥 **Vera Rubin/NVL 系列路线仍被市场盯住 CPO 与 rack-scale midplane 风险。** Tom's Hardware 7 月报道转述 Huang 表态称 Rubin roadmap intact，同时也汇总了外部关于 Kyber NVL144、NVL576 和 CPO 技术挑战的传闻；这些不是官方规格，但反映 rack-scale system manufacturing 与 optical I/O readiness 已成为投资者关注点。来源：[Tom's Hardware Vera Rubin report](https://www.tomshardware.com/tech-industry/artificial-intelligence/nvidias-huang-vows-to-deliver-giant-amounts-of-vera-rubin-company-says-that-our-roadmap-is-intact)
3. ⭐ **UALink 白皮书把 scale-up accelerator fabric 定义在最多 1,024 endpoints 的 rack-scale pod。** 白皮书提到 memory semantics、64-256 byte transactions、57-bit physical address space、SAI 管理和 93% effective bandwidth efficiency 目标；对 NVLink 之外的开放 scale-up 生态有直接参考价值。来源：[UALink white paper PDF](https://ualinkconsortium.org/wp-content/uploads/2026/01/UALink_White_Paper_Publication_Candidate_FINAL_VERSION.pdf)
4. ⭐ **OCP AI scale-up network 讨论继续推动开放硬件视角。** OCP 相关公开活动把 AI scale-up networks、low-latency fabric 和 accelerator cluster 作为开放系统议题；应与 UALink、Ultra Ethernet、NVLink Fusion 共同跟踪。来源：[OCP AI Scale Up Network video](https://www.youtube.com/watch?v=nfGMqLUrMnw)

## 📄 论文速递

1. 🔥 **arXiv:2602.12521：Opus: Photonic Rail-Optimized Fabric in ML Datacenters。** 论文提出用 reconfigurable optical circuit switches 重构 ML datacenter rail fabric，减少 OEO 转换与 datapath switch ASIC 能耗，适合关注 scale-out 光交换。来源：[arXiv:2602.12521](https://arxiv.org/html/2602.12521v3)
2. 🔥 **arXiv:2603.21313：3D optoelectronics and co-packaged optics。** 论文强调 CPO/3D photonic integration 不是单器件优化，而是 packaging、thermal management、serviceability 和 system-level robustness 的架构承诺。来源：[arXiv:2603.21313](https://arxiv.org/abs/2603.21313)
3. 🔥 **arXiv:2606.14052：Nonlinear Pluggable Optics。** 论文提出 DSP-free NLPO transceiver，用 photonic NGRC 做 IM/DD distortion compensation，目标是在 LPO 低功耗/低延迟与 DSP-based reach/robustness 之间找折中。来源：[arXiv:2606.14052](https://arxiv.org/html/2606.14052)
4. ⭐ **arXiv:2602.08284：2.5D co-packaged optical I/O chipsets on a SiON/Si interposer。** 论文实验展示 4 x 100G optical interconnection，使用 SiON/Si interposer、InP active devices 和 WDM routing，对 flexible optical I/O packaging 有参考价值。来源：[arXiv:2602.08284](https://arxiv.org/abs/2602.08284)
5. ⭐ **arXiv:2604.12566：Scalable 3D silicon nitride photonic interposer。** 论文面向 CPO/scale-up compute clusters，关注 3D SiN photonic interposer 如何缓解高密度互连的 routing 与功耗瓶颈。来源：[arXiv:2604.12566](https://arxiv.org/html/2604.12566v1)
6. ⭐ **arXiv:2605.28971：Micro-Transfer Printing of Lithium Niobate on 200 mm Silicon Photonics。** 论文讨论 TFLN 到 SiPho 平台的 wafer-scale heterogeneous integration，用于高速度、低能耗 photonic interconnects。来源：[arXiv:2605.28971](https://arxiv.org/html/2605.28971v1)
7. 📌 **arXiv:2607.08407：Who Needs DRAM? We Have Fiber。** 论文提出利用 optical fiber network 作为 memory-like sharing substrate，虽偏架构探索，但反映 CPO/光互连已被用于重新思考 LLM 权重复用和数据复制成本。来源：[arXiv:2607.08407](https://arxiv.org/html/2607.08407v1)
8. 📌 **arXiv:2604.15261：RNG: Flat Datacenter Networks at Scale。** 论文介绍 flat datacenter network、quasi-random graph 与 passive optical device，用于降低 cabling complexity 和网络成本，可作为 AI data center scale-out 拓扑参考。来源：[arXiv:2604.15261](https://arxiv.org/html/2604.15261v2)
