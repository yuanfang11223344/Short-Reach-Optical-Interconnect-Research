## 文件命名

日报：

```text
YYYY-MM-DD.md
```

周报：

```text
weekly-YYYY-WXX.md
```

其中 `YYYY-WXX` 使用 ISO week 编号。

## 目录映射

| 报告类型 | Obsidian | Learning_repo | 专题 Git 仓库 |
|---|---|---|---|
| 日报 | `output/YYYY-MM-DD.md` | `YYYY-MM-DD.md` | `报告/日报/YYYY-MM-DD.md` |
| 周报 | `output/weekly-YYYY-WXX.md` | `weekly-YYYY-WXX.md` | `报告/周报/weekly-YYYY-WXX.md` |

## 同步一致性要求

同一个日期或同一个周次的三份 Markdown 报告，正文内容必须一致。如果发布后修正错字、来源链接或技术表述，必须在同一次维护中同步更新三处副本。

## 执行报告格式

每次生成或同步后，执行报告需要包含：

- 模式：日报或周报；周一 10:00 触发时应同时说明日报和周报。
- 检索方向或信源数量。
- 选入事件/条目数量。
- 去重来源链接数量。
- 保存路径。
- 专题 Git 仓库的 commit hash 和推送分支。
